## 1. プロジェクト概要

**RustComputers** は Minecraft 1.20.1 (Forge 47.4.10) の Mod。  
ゲーム内にコンピューターブロックを設置し、Rust (no_std) で書いたプログラムを WASM として実行できる。  
計算エンジンには **Chicory 1.7.2** (Pure Java WASM Runtime + JIT Compiler) を採用。

### ゴール
- CC:Tweaked の Lua VM を Rust/WASM で代替する
- CC:Tweaked の Java ペリフェラルエコシステムとの互換性を維持する
- async/await によるノンブロッキングな API を提供する

## 4. アーキテクチャ

### 全体フロー

```
[Rust Program (.rs)]
    ↓ cargo build --target wasm32-unknown-unknown
[.wasm binary]
    ↓ Upload via GUI (UploadWasmPacket)
[WasmEngine.start()]
    ↓ Chicory Parser.parse() → Instance.builder() → JIT Compile
[wasm_init()]  → executor::init(main_future)
    ↓
[毎 tick: wasm_tick()]  → executor::tick() → Future.poll()
    ↓                       ↑
[host_log, host_stdin_read_line, host_request_info, ...]
    ↓
[Java HostFunctions → WasmEngine callbacks → WASM Memory R/W]
```

### Rust → Java 通信

| Rust 関数 | FFI | Java HostFunction | 同期/非同期 |
|---|---|---|---|
| `rc::println!()` | `host_log(ptr, len)` | `hostLog` | 即時 |
| `rc::read_line()` | `host_stdin_read_line(result_ptr, buf_sz)` | `hostStdinReadLine` | 非同期 |
| `peripheral::request_info()` | `host_request_info(periph_id, method_id, ...)` | `hostRequestInfo` | 非同期 |
| `peripheral::do_action()` | `host_do_action(periph_id, method_id, ...)` | `hostDoAction` | 非同期 |
| `peripheral::request_info_imm()` | `host_request_info_imm(...)` | `hostRequestInfoImm` | 即時 |
| 内部 poll | `host_poll_result(req_id, written_ptr)` | `hostPollResult` | 即時 |
| `io::computer_id()` | `host_get_computer_id()` | `hostGetComputerId` | 即時 |
| `io::is_mod_available()` | `host_is_mod_available(mod_id)` | `hostIsModAvailable` | 即時 |

### 非同期モデル

1. Rust 側: `no_std` ミニ Executor。`wasm_tick()` → `executor::tick()` → 1 回 `poll`
2. 非同期ホスト関数は `request_id` を返す
3. `RequestFuture.poll()` が毎 tick `host_poll_result(req_id)` を呼ぶ
4. Java 側: `PendingResult` に結果書き込み → 次の poll で `Ready` 返却

## 5. ビルド手順

### Rust クレート → WASM

```bash
cd crates/rust-computers-api
cargo build --example hello --target wasm32-unknown-unknown --release
# 出力: target/wasm32-unknown-unknown/release/examples/hello.wasm
```

### Forge Mod → JAR

```bash
cd forge
./gradlew build
# 出力: build/libs/rustcomputers-<version>.jar
```

jarJar によって Chicory (runtime, wasm, compiler) が mod JAR に同梱される。

---

## 6. WASM デプロイ方法

1. `.wasm` ファイルをビルド
2. ゲーム内でコンピューターブロックを設置
3. 右クリックで GUI を開く
4. GUI から `.wasm` ファイルをアップロード（UploadWasmPacket）
5. "Run" ボタンで実行開始

ファイル保存先: `<world>/rust computers/computer/<id>/`

## 9. 今後の展望

### JIT / AOT コンパイル対応

現在 Chicory をインタープリタモードで動作させている。Chicory 1.7.2 には
Runtime Compiler (`chicory:compiler` アーティファクト) が存在するが、
Forge/Minecraft が内部で使用する ASM バージョン (9.x) と Chicory Compiler が
要求するバージョンが競合するため、現時点では除外している。

将来的に以下のいずれかで解決し、JIT または AOT ビルドへ移行することで
実行速度を大幅に改善できる見込み。

| アプローチ | 概要 | 難易度 |
|---|---|---|
| **Forge Classpath 分離** | `forgegradle` の `jarJar` または classloader 分離で Chicory Compiler を隔離 | 中 |
| **Chicory 上流バグ修正待ち** | Chicory が ASM バージョンを柔軟化した場合、`chicory:compiler` を再度 `runtimeOnly` で追加するだけで有効化 | 低 |
| **AOT 事前コンパイル** | `.wasm` → Java bytecode 変換を Gradle タスクとして mod ビルド時に実行し、クラスファイルとして同梱 | 高 |

> **参考**: インタープリタ vs JIT の実行速度比は一般的に 5〜20倍。
> 複雑な計算を行うプログラムで体感できるレベルの差が生じる可能性がある。
